# Spike N — native cart edit sequence (n1-n6).
#
# Format: <edit_id> <expected_outcome> <description>
#   expected_outcome:
#     PASS                  — buffer+digest round-trip succeeds
#     PASS-LATENCY          — as PASS but also gate on latency < 3000 ms
#     FAIL-WITH-DIAGNOSTIC  — expected failure; stderr byte-equal
#
# Each edit is a PRE→POST pair:
#   PRE  = save side ELF  (-DEDIT_SIDE=0)
#   POST = load side ELF  (-DEDIT_SIDE=1)
#
# The harness runs:
#   1. PRE ELF for S frames → BUFFER line
#   2. POST ELF loads BUFFER → BUFFER (post-migration) + DIGEST stream
#   3. Compare post-migration BUFFER cross-host (SHA-256 must match)
#   4. Compare DIGEST stream cross-host (SHA-256 must match)
#   5. Compare DIGEST stream to same-host POST full-run suffix from S+1
#   6. Check runtime_ms + packer_ms < 3000
