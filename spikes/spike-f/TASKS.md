- [x] `make lua-src`: fetches Lua 5.4.7 from `https://www.lua.org/ftp/`,

- [x] `host.c`: `luaL_newstate` + `luaL_openlibs`, embedded-file loader

      Frame counts (`doom_tick=30`, `entity_update=50`, others=20,
      `doom_tick_gc=30`) match Spike B's `BENCH_FRAMES_*` defaults so

      `host.c` and the results doc.
- [x] `make lua-wasm`: builds `build/lua.{js,wasm}` (~275 KB wasm + 77

## Architecture follow-up (Spike F passed performance)

- [ ] Open ADR addendum or new ADR — per-target execution model.
      Touches ADR-0007 (structural determinism), ADR-0025 (Lua cart
      execution model), ADR-0038 (two-layer sandboxing). Decision
      shape: WASM target uses Lua-direct-to-WASM; hardware and
      native targets continue to use RV32IMFC + rv32emu.
- [ ] Determinism cross-check vs Spike D once D produces reference
      hex digests. Expect divergence (different libm / different
      `printf("%g", float)` paths); document the divergence and
      its consequences for save-state / replay / netplay on the
      WASM target.
- [ ] Security-model write-up: how `_ENV` (ADR-0066) plus the
      ADR-0079 standard-library allowlist reproduce the RV32IMFC
      sandbox's guarantees on the WASM target, and where the
      asymmetry is load-bearing. Spike F does not test this; the
      write-up is informed by the existing ADRs, not by new
      measurement.
- [ ] Stdlib trim: bring forward Spike B's `lua_init_libs.c` (base +
      table + string + math, omit io / os / loadlib / coroutine /
      debug / utf8) into the production WASM build per ADR-0079.
      One-file change; does not affect the timing numbers.
